# FloBook
FloBook是一個工程分析計算資訊平台，匯集工程分析計算過程中的數據，這些數據能被搜尋、統計、並經過智能化判斷，提供使用者於計算工作執行時所需的重要資訊。

## Introduction
### FloBook Monitor (軟、硬體用量監控平台)
以先進的監控技術，同時針對多部計算機器與軟體授權用量進行即時與歷史數據的監控。透過這個平台：
- 可得知計算機器的即時用量、歷史用量與稼動率。
- 可得知軟體授權的即時用量、歷史用量與稼動率。
- 可辨識個別使用者的計算資源占用率。

功能規格包含：
1. 矩陣式即時資源監控板(Matrix Monitor)
藉由矩陣型式的資料方塊來呈現不同受控設備的各種資料，包含CPU、RAM的即時使用率、硬碟空間的使用率、已登入此受控設備的使用者數量(以Windows Server 2012為例)以及正被占用的ANSYS License modules數量。Matrix Monitor是一個互動式的圖表，將滑鼠移動到您感興趣的方塊上，您可以看到一些較為詳細的資料細節，如已登入此受控設備的使用者清單或是正被占用的ANSYS License modules名稱。
2. 儀表式即時資源監控板(Gauges Monitor)
數位儀表簡單易讀，適合用來快速瀏覽不同受控設備的即時資源用量，包含CPU、RAM的即時使用率、硬碟空間的使用率，可與矩陣式即時資源監控板交互使用，各有優點。
3. 即時計算機器硬體用量詳情
FloBook Monitor可以擷取受控設備的即時硬體資料以呈現更多細節，例如CPU的剩餘核心數、RAM的剩餘容量、硬碟的剩餘空間等，還可以在多人同時登入的使用情境下顯示每個使用者各自的資源使用量，讓您可以清楚掌握硬體資源的分配狀況。
4. 即時軟體授權用量詳情
FloBook Monitor對ANSYS License modules的控管有良好的支援，可以顯示該受控設備License manager中的所有License modules總量、即時使用量、License module的占用時間與使用者，讓您可以清楚掌握所有ANSYS License module的使用狀況，以擬定資源分配對策、計算排程，或是釐清軟體添購的需求。
5. 計算機器歷史用量報表 
FloBoook Monitor除了顯示受控設備的即時資料，還可以查閱過去一段時間內的硬體資源稼動率，可以呈現不同使用者在該時間區間內使用的計算資源。
6. 軟體授權歷史用量報表
同樣的，對於ANSYS License modules，FloBook Monitor也能精確地將該模組在一段時間內的使用時數調閱出來，可以觀察不同模組被哪些使用者使用，也可以觀察這些使用者都用了哪些模組。

### FloBook Solve (雲端排程運算平台)
FloBook Solve可讓您個人或是您的團隊進行計算排程，不同於市面其他的排程軟體，FloBook Solve會考慮ANSYS License模組的可用數量，如此一來便能防止排程軟體在取得CPU核心數量許可之後因為ANSYS License模組數量不夠而無法運算甚至意外佔據計算資源的問題。

功能規格包含：
1. 支援高效能叢集式運算
支援ANSYS HPC、ANSYS HPC Pack讓高運算量的模擬分析變得可行，加速求解時間，提供高保真模型解決方案，縮短設計評估週期，增加設計方案數量，並且支援以Workbench統合結構、流體、電磁等多物理耦合的求解工作。
2. 支援考量ANSYS License數量的任務排程
除了支援考量計算機器CPU剩餘核心數的排程以外，FloBook也會根據ANSYS License Modules的剩餘數目來決定計算任務是否執行。
3. 支援GPU的排程選項
若您的計算機器具備GPU，FloBook Solve加入了GPU的排程選項，讓您的GPU剩餘數量也可以納入排程的參考。
4. 自訂任務優先順序
(1)	針對不同的計算任務，可以開啟檔案位置查看計算結果、終止計算任務並釋放ANSYS License資源、查看即時計算狀況、結案並從佇列中移除此任務。
(2)	透過直覺的排程表格對佇列中的計算任務進行排序。
(3) 針對需要大量CPU核心數的計算任務，可以設定為最高優先，其他需要較少核心數的計算任務便會等待最高優先的任務完成後，才開始執行，這樣可以避免大型任務因核心數不足無法開始運算的問題。
5. 支援簡易硬體即時監控視窗
設定受控的計算機器後，可以藉由一個簡易的監控視窗來即時監控該計算機器上CPU與RAM的使用率。


### FloBook Track (分析專案知識庫與追蹤管理系統)
針對CAE團隊而研發的知識庫系統，可以詳細記錄不同專案的分析目標、相關檔案、成果報告，讓CAE專案與它的解決過程得以建檔，日後使用者能藉由所有可能的關鍵字來搜尋到這個專案來提供後人參考、傳承知識，讓所謂的「經驗法則」變得有跡可循。

功能規格包含：
1. 知識庫系統(專案建立、專案搜尋)
透過表單方式填寫專案詳細資料，例如為您的專案建立ANSYS版本號、領域、執行人等關鍵字，日後可讓搜尋系統透過這些關鍵字找到這個專案。FloBook Track提供一個彈性十足的資料表格，當您搜尋某個關鍵字，所有與之相關的專案都會被篩選出來，不限於搜尋特定欄位，上述ANSYS版本號、領域、執行人等都可以作為搜尋的關鍵字。
2. 專案細節瀏覽
您可以點擊「查看詳情」進入專案細節頁面，瀏覽關於這項專案的所有相關內容，若權限許可，您可以在這裡修改專案內容、上傳相關檔案。必要的時候，您可以設定該專案的瀏覽權限，限制可瀏覽該專案的成員。
3. 專案追蹤系統
FloBook Track提供查核點定義與專案事件紀錄等功能，讓該專案的團隊成員可以自訂查核項目、追蹤專案進度、查看誰修改了專案描述或是誰上傳了新檔案。
4. 檔案管理(存放、預覽、下載)
FloBook Track提供一個檔案預覽的功能，您可以在這裡瀏覽該專案的所有相關檔案，可以做檔案順序的更改、檔案下載或檔案刪除。檔案種類支援圖片檔、PPT、PDF、.txt與ANSYS的.ans這類文字檔等。
